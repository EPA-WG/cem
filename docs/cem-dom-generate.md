# Task: Generate Nx TypeScript library + native Node CLI for `@epa-wg/cem-dom`

Build or complete the package `@epa-wg/cem-dom` as a publishable TypeScript ESM npm package that exposes both:

1. a library API for parser/validator usage, and
2. a CLI binary named `cem-dom`.

The package should support direct TypeScript execution during development using native Node TypeScript support, without
`ts-node`, `tsx`, Babel, Jest, or Vitest.

Assume latest Node.js.
The test command should invoke native node test runner. 
The test coverage should be generated by native node test runner.
The usual target for TypeScript projects like `lint` or `typecheck` should be supported.
The library build should produce ESM output.
Package type should be `module`.

---

## Primary goals

Create a clean library/CLI package with this intent:

```txt
@epa-wg/cem-dom
  library API:
    parseCemDom(...)
    validateCemDom(...)
    formatDiagnostics(...)

  CLI:
    cem-dom parse <file>
    cem-dom validate <file>
    cem-dom version
    cem-dom help
